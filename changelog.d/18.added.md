Background (async) runs. `POST /threads/{id}/runs` now starts the graph in a background task and returns immediately with a `pending`/`running` run. New endpoints to consume them: `POST /threads/{id}/runs/wait` (run to completion, returns the final state values), `GET /threads/{id}/runs/{rid}/join` (wait for a terminal state), `POST /threads/{id}/runs/{rid}/cancel` (`action=interrupt|rollback`, optional `wait`), and `DELETE /threads/{id}/runs/{rid}`. The `rollback` and `interrupt` multitask strategies are now implemented (cancel — and, for rollback, delete — the active run) instead of all returning 409 like `reject`.
